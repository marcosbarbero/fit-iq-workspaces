## ğŸ’¡ Key Insights

### 1. Different Data Sources â†’ Different Flows

**Text Input:**
- Raw text has NO nutrition data
- MUST be processed by backend
- WebSocket is the ONLY way to get results

**Photo Input:**
- Image HAS embedded nutrition data (visual)
- Backend processes DURING upload
- Results available IMMEDIATELY in response

### 2. Idempotency Matters

**Photo Recognition:**
- Backend already processed the image
- Backend already calculated nutrition
- Backend already stored the data
- Confirmation = "Yes, this is correct" (not "process this")

**Text Input:**
- Backend hasn't seen the data yet
- Submission = "Process this text"

### 3. User Expectations

**95% of photo uploads:**
- "Looks good, save it!"
- No changes made
- Expect instant confirmation

**5% of photo uploads:**
- "Hmm, that quantity looks wrong"
- User makes edits
- Expect reprocessing (acceptable)
